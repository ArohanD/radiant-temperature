## How to edit and preview

The Marp live-preview server picks up edits to `presentation.md` and figure PNGs on save:

```bash
# from /home/arohan/Documents/code/radiant-temperature  (repo root)
npx --yes @marp-team/marp-cli@latest --html --server . --allow-local-files
# then open http://localhost:8080/slides/presentation.html
```

**Important gotcha:** the server must be run from the **repo root** (not `slides/`), because the markdown references figures via `../figures/durham_hayti/slides/…` — paths that resolve outside the `slides/` directory.

Final PDF for distribution:

```bash
npx --yes @marp-team/marp-cli@latest --html slides/presentation.md --pdf -o slides/presentation.pdf --allow-local-files
```

(Marp emits one format per invocation; run twice for both PDF and HTML.)

## Figures

All slide figures are generated by `scripts/08_make_slide_visuals.py`. Outputs land in `figures/durham_hayti/slides/`. To regenerate just one:

```bash
./env/bin/python -c "
import sys; sys.path.insert(0, 'scripts')
import importlib.util
spec = importlib.util.spec_from_file_location('m', 'scripts/08_make_slide_visuals.py')
m = importlib.util.module_from_spec(spec); spec.loader.exec_module(m)
m.fig_study_site()  # or any other fig_* function
"
```

Or all at once: `./env/bin/python scripts/08_make_slide_visuals.py`

## Recent decisions (so you don't undo them)

These were deliberate and should not be reverted:

1. **`study_site.png` right panel zooms to the densest 800 m × 800 m cluster** (170 of 245 plantings), not the full 2 km AOI. Helper `_densest_cluster_center()` in the same file. The AOI red box is intentionally omitted from this panel since the zoom is fully inside it; the title and legend make the relationship explicit.
2. **All scale bars in slide figures are zebra-style** (4 segments, thin, endpoint labels, white halo) via the upgraded `_add_scalebar()` helper. The standalone `topdown_tmrt_peak.png` cartographic deliverable kept its detailed scale bar — it's not in the deck.
3. **Tick formatter on UTM-coordinate figures**: `MaxNLocator(nbins=5)` + `ScalarFormatter(useOffset=True, scientific=False)` so 6/7-digit eastings don't run together at slide size.
4. **LLM-style giveaways scrubbed from slide-rendered text** (em-dashes, "isn't X, it's Y", "actually" hedges). Speaker notes left untouched. Em-dashes preserved only where typographically correct: numeric/date ranges (2025–2028, 38–46 %) and the actual paper title in the Lindberg 2008 citation.
5. **Em-dashes also scrubbed from figure titles, suptitles, and axis labels** in `08_make_slide_visuals.py`. Replaced with `:` for title-subtitle patterns and `,` or `.` for asides.
6. **The Pickering / Mistry / Lindberg (2023) reference was fabricated** and has been removed from both the references slide and the speaker note on slide 10. The DOI `10.1016/j.uclim.2023.101517` actually resolves to a paper by Zhang et al. about Hong Kong urban warming. The slide-10 speaker note now credits "the original Lindberg/Holmer/Thorsson SOLWEIG papers and subsequent UMEP studies" instead.
7. **The Kamath et al. (2026) JOSS citation was paraphrased** and has been corrected to match the actual title: *SOLWEIG-GPU: GPU-Accelerated Thermal Comfort Modeling Framework for Urban Digital Twins*, JOSS 11(118), 9535. Full author list: Kamath, Sudharsan, Singh, Wallenberg, Lindberg, Niyogi.
8. **Conclusion slide bullets** were rewritten (option C from a brainstorm). Bullet 2 reframed as "Method" (a data-fusion pipeline, not a "contribution"). Bullet 3 rewritten as full sentences instead of dot-separated fragments.
